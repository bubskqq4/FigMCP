## Recommended Workflow

### 1. Design Phase
- Use Figma with design system plugins
- Extract colors with ColorZilla
- Check accessibility with Stark
- Export design tokens with Design Tokens plugin

### 2. Development Phase
- Use Figma MCP Server design system tools
- Get guidelines: `get-design-guidelines()`
- Get style inspiration: `get-style-guide()`
- Generate components with Figma to Code plugins
- Use Tailwind IntelliSense in VS Code

### 3. Quality Assurance
- Test accessibility with axe DevTools
- Check performance with Lighthouse
- Visual regression with Chromatic
- Cross-browser testing

### 4. Documentation
- Generate design docs: `generate-design-docs()`
- Export design system: `export-design-system()`
- Create Storybook for component library
- Use zeroheight for team documentation


## Tool Selection Criteria

When choosing design tools, consider:

1. **Integration** - Does it work with Figma/MCP?
2. **Export Options** - Can it export to your stack?
3. **Team Collaboration** - Multi-user support?
4. **Cost** - Free tier available?
5. **Learning Curve** - How quickly can team adopt?
6. **Maintenance** - Actively maintained?
7. **Community** - Good documentation and support?

## Figma MCP Specific Tools

### Use Built-in MCP Tools

The Figma MCP Server includes 200+ tools. For design work, focus on:

**Design System Tools (NEW!):**
- `get-design-guidelines` - Get comprehensive design guidelines
- `get-style-guide` - Get curated style guides
- `get-style-guide-tags` - Explore design tags
- `create-design-guideline` - Create custom guidelines
- `create-style-guide` - Create custom style guides
- `add-design-tag` - Add custom tags
- `export-design-system` - Export your design system
- `generate-design-docs` - Generate documentation

**Variable/Token Tools:**
- `get-variables` - Get all variables in file
- `set-variables` - Set design tokens
- `create-variable` - Create new variables
- `update-variable` - Update existing variables

**Component Tools:**
- `create-component` - Create reusable components
- `get-all-components` - List all components
- `create-instance` - Use component instances
- `add-component-property` - Add props to components

**Style Tools:**
- `create-text-style` - Create text styles
- `create-effect-style` - Create effect styles
- `get-text-styles` - List all text styles

**Screenshot Tools:**
- `take-screenshot` - Export node as image
- Validate designs visually

## Tool Maintenance

### Keep Tools Updated

```bash
# Update Figma plugins regularly
# Check for updates in Figma > Plugins > Manage plugins

# Update VS Code extensions
# Extensions > ... > Check for Updates

# Update npm packages
npm outdated
npm update

# Audit dependencies
npm audit
npm audit fix
```

### Tool Hygiene

- Remove unused Figma plugins (slows down Figma)
- Disable VS Code extensions you don't use
- Clear cache periodically
- Keep design file organized
